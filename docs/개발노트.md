# 힉스노티 개발 노트

> 이 프로젝트가 어떻게 시작됐고 각 결정이 왜 내려졌는지 기록. (2026-07-02 첫 정리)

## 프로젝트 결정

- **이름**: 힉스노티 (Higgs Noti). "Higgsfield + Notify" 합성. 리포/폴더 슬러그는 URL 호환을 위해 ASCII `higgs-noti`, 표시 이름은 "힉스노티".
- **성격**: 영상 제작 프로젝트(REBUILD)와 분리된 **별도 프로젝트**. 힉스필드 영상 생성 완료를 알려주는 **브라우저 유저스크립트** 하나짜리 단일 도구.
- **범위**: 힉스필드 생성 완료 알림 **단일 기능**에 집중. (향후 여러 AI 웹서비스 도구를 모으는 상위 사이트 **"AI Field"** 는 별도 브랜드로 남겨둠 — [../TODO.md](../TODO.md) 미래 계획 참고.)
- **공개 계획**: GitHub(`happybinnyy`) 공개 + **GreasyFork** 배포(자동 업데이트) + **후원(도네이션)**.
- **라이선스**: MIT. **"비공식(unofficial), 제작사와 무관"** 명시 (상표 이슈 회피).
- **현재 상태**: 로컬 git 저장소만 존재. GitHub 공개는 이 프로젝트에서 진행 예정([../TODO.md](../TODO.md)).

> **개명 이력 (2026-07-02)**: 원래 "AI필드(AIfield) — 여러 AI 유저스크립트 모음집" 컨셉으로 시작했으나, 알림 도구 하나에 집중하기로 방향을 바꿔 **"힉스노티(higgs-noti)" 단일 프로젝트**로 재편성. 구조를 평탄화(하위 폴더 → 루트)하고 브랜드·URL을 전부 통일했다. "AI Field"라는 상위 모음집 아이디어는 미래 계획으로만 보존.

## 첫 도구: higgs-noti — "완료 감지 신호" 찾은 과정

목표: 힉스필드 영상 생성이 **완료될 때만** 알림 (시작할 때 울리면 안 됨).

여러 방식을 시도했고 대부분 실패 → 폐기했다. 이 기록이 유지보수·다른 사이트 도구 만들 때 참고가 된다.

| 시도한 신호 | 폐기 이유 |
|------------|----------|
| "All assets" 옆 숫자 | My generations 화면엔 그 텍스트가 없어 `null`. 프로젝트 화면엔 있지만 시작/완료 판별 불가 |
| "New" 배지 개수 증가 | 완료가 아니라 생성 **시작** 시 붙음 → 시작하자마자 오탐. nav 고정 "New" 4개도 노이즈 |
| 스피너 `.animate-spin` | 생성 중이 아닐 땐 0이라 처음엔 "미사용"으로 오해. (실제 생성 중 타일엔 존재) |
| `<video>` 개수 | 마우스 호버 미리보기로 0↔1 토글 → 완료와 무관 |
| `<img>` 개수 | 목록 **가상화(virtualization)** 로 스크롤·로딩마다 들쭉날쭉 |
| "Processing" 텍스트 개수 | 생성 도중 텍스트가 단계 전환으로 사라짐 → **완료 전에** 오탐 |

**최종 채택: `data-job-status` + "화면 안정성" 기반 완료 감지 (v1.0.3)**

생성 중 타일의 실제 DOM(직접 캡처로 확인):
```html
<div data-asset-id="..." data-job-status="queued" ...>
  ... <span>Processing</span> ...
  <button ...>Cancel</button>
  <svg class="animate-spin">...</svg>
</div>
```
- `data-job-status` 는 힉스필드의 **작업 상태 공식 속성**. 진행중=`queued`/`processing` 등, 완료=`completed`. 모든 타일에 고유 `data-asset-id` 보유.

**개수 감소 방식이 실패한 이유 (2026-07-09 실측으로 교정)**
- 단순 "진행중 개수 감소 = 완료"(v1.0.0)는 **화면 전환에서 오탐**: 좌측 폴더/프로젝트를 이동하면 진행중 타일이 통째로 사라져 개수가 줄고 → 완료로 오인해 알림이 울린다.
- "같은 id가 완료로 전환"만 보는 방식(v1.0.2)은 **완료를 놓침(미탐)**: 완료 시 타일 id가 유지되지 않고 **새 id로 교체되는** 경우가 있어 매칭이 안 된다.
- **최종(v1.0.3)**: 진행중이던 타일이 사라진 순간, "화면이 그대로인지"로 판정한다.
  직전의 비(非)진행중 타일들(완료본)이 지금도 **절반 이상 그대로 남아있으면 = 같은 화면 → 완료로 인정**,
  그리드가 통째로 바뀌었으면(겹침 급감) **= 폴더 이동/스크롤 → 무시**.
- 실제 생성 완료로 검증: `진행중 1 → 0, stable=true, finished=1` 로그 확인 + 알림 정상. 폴더 이동 시 무음 확인.

## 핵심 함정 (gotcha)

- **유저스크립트 실행 권한** (스크립트가 안 돌던 **실제 원인**): Tampermonkey 5.3+ 는 크롬 계열에서 실행 권한을 따로 켜야 하고, 안 켜면 설치해도 **아예 실행되지 않는다.** 브라우저별로 다르다:
  - **크롬/엣지 138+**: `chrome://extensions` → Tampermonkey 세부정보 → "사용자 스크립트 허용" ON
  - **크롬/엣지 구버전**: `chrome://extensions` → "개발자 모드" ON
  - **웨일(Whale)**: "사용자 스크립트 허용" 토글이 **아예 없다.** → 일반 Tampermonkey(5.x)로는 구동 불가. **Tampermonkey Legacy(MV2)** 를 설치하면 이 권한 없이 동작한다. (실제로 웨일 환경에서 이 방법으로 해결)
  - 새 도구 배포 README마다 이 안내를 넣을 것.
- 힉스필드 목록은 **가상화**(`will-change-transform` 가상 행) + `qc-` 접두어 디자인 시스템. → **개수 세기 방식(img/tile count)은 신뢰 불가**, 상태 속성/명시적 표시를 써야 한다.
- 마크다운 코드블록을 복사할 때 백틱(```)이나 헤더가 딸려 들어가 오류가 났었음 → 배포 스크립트는 복사 버튼/원본 파일에서 받도록 안내.
