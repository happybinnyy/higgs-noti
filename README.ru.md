# Higgs Noti

[English](README.md) | [한국어](README.ko.md) | [简体中文](README.zh-CN.md) | **Русский** | [Español](README.es.md) | [Português (BR)](README.pt-BR.md) | [日本語](README.ja.md)

Пользовательский скрипт для браузера, который уведомляет о завершении генерации видео на Higgsfield ([higgsfield.ai](https://higgsfield.ai)).
Время генерации каждый раз разное, поэтому больше не нужно постоянно следить за вкладкой.

> ⚠️ **Неофициальный** инструмент. Никак не связан с Higgsfield.
> Он лишь считывает страницу (DOM), чтобы показать уведомление, — не собирает и не распространяет контент и не обходит оплату.

## Возможности

- **Баннер на странице** — не требует разрешений, виден всегда
- **Звук** — воспроизводится после того, как вы хотя бы раз кликнули по странице (политика автозапуска браузера)
- **Уведомление на рабочем столе** — если разрешены уведомления ОС/браузера
- **Метка ✅ в заголовке вкладки** — заметна, даже когда вы на другой вкладке
- **Панель статистики** — кнопка 📊 (внизу справа): количество за сегодня/неделю/всё время, статистика времени генерации, распределение по дням недели и часам, экспорт в CSV

## Установка

1. Установите [Tampermonkey](https://www.tampermonkey.net/) в браузере
2. ★ **Включите разрешение на выполнение пользовательских скриптов** — без этого скрипт **вообще не запустится**, даже после установки. Способ зависит от браузера:
   - **Chrome / Edge (138+)**: `chrome://extensions` → **Сведения** о Tampermonkey → включите **«Разрешить пользовательские скрипты»**
   - **Chrome / Edge (старые версии)**: `chrome://extensions` → включите **«Режим разработчика»** (вверху справа)
   - **Whale**: переключателя «Разрешить пользовательские скрипты» **вообще нет**, поэтому обычный Tampermonkey (5.x) не может запускать скрипты → установите **Tampermonkey Legacy** (MV2) из магазина; он работает без этого разрешения.
3. Откройте [`higgs-noti.user.js`](higgs-noti.user.js) для установки → обновите вкладку Higgsfield (F5)
4. Если вверху справа появился баннер `🎬 Notifier running (active N)` — всё работает

## Как это работает

Плитки, которые генерируются, имеют **атрибут статуса задачи**, например `data-job-status="queued"`/`"processing"`.
Когда «выполняющаяся» плитка исчезает, уведомление отправляется **только если остальной список (вид) остался прежним** — то есть это реальное завершение.
Если весь список изменился из-за переключения папок или прокрутки, это отличается от завершения, поэтому ложных срабатываний нет.

## Устранение неполадок

- **Нет даже баннера** → отключено «Разрешить пользовательские скрипты» или скрипт выключен.
  Проверьте в консоли (F12) лог `[힉스알림] 시작, 진행중 = N`.
- **Отсутствует только уведомление ОС** → отключите «Фокусировку внимания» Windows и установите уведомления сайта на «Разрешить».
  (если баннер/звук работают, значит сам скрипт в порядке)
- **Завершение не определяется** → Higgsfield изменил атрибут/значение `data-job-status`.
  Проверьте (F12) генерируемую плитку, узнайте фактический атрибут и измените регулярное выражение/селектор `ACTIVE` в скрипте.
